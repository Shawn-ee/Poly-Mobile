```bash
DEPOSIT_CONFIRMATIONS="20"
DEPOSIT_MIN_USD="2"
DEPOSIT_WALLET_ENCRYPTION_KEY="replace_with_64_char_hex_secret"
TREASURY_WALLET_ADDRESS="0xTreasuryWallet"
TREASURY_PRIVATE_KEY="0x..."
DEPOSIT_MONITOR_POLL_INTERVAL_MS="15000"
```

Generate a local encryption key with:

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

Notes:
- `POLYGON_RPC_URL` is the Polygon RPC endpoint, not a wallet address.
- `POLYGON_USDC_ADDRESS` is the Polygon USDC token contract, not your deposit wallet.
- `DEPOSIT_WALLET_ENCRYPTION_KEY` is created by you, must be 64 hex characters, and must never be committed.
- User deposit addresses are generated by the backend.
- The deposit monitor watches Polygon USDC `Transfer` events to those generated user deposit addresses.
- For new production-facing deposits, this app uses native Circle USDC on Polygon: `0x3c499c542cEF5E3811e1192ce70d8cC03d5c3359`.

## Reference Market Runbook

Use three terminals for local reference-market operations:

Terminal 1:
```bash
cd Poly
npm run dev
```

Terminal 2:
```bash
cd Poly
npm run reference:snapshot-watch
```

Terminal 3:
```bash
cd ../poly-bot
npm run liquidity:runtime
```

Notes:
- `reference:snapshot-watch` keeps `ReferenceQuoteSnapshot` fresh every 5 seconds and does not place or cancel orders.
- `liquidity:runtime` is dry-run by default. It only manages live quotes when `SYSTEM_LIQUIDITY_DRY_RUN=false`, `LIVE_SYSTEM_LIQUIDITY_ENABLED=true`, and the market lifecycle is explicitly `live_enabled`.
- The market detail page polls `/api/markets/[id]/reference` every 5 seconds, so the stale warning should clear while the watcher is running.

Useful admin flow:
- Seed a market bot from `poly-bot`:
```bash
npm run liquidity:seed-market-bot -- --slug will-france-win-the-2026-fifa-world-cup-924 --capitalDollars 1000 --mintDollars 200 --dryRun true
```
- Mark lifecycle states from `/admin/reference-markets`
- Pause or emergency-stop from `/admin/reference-markets`
- Cancel bot quotes with the admin `cancel_bot_quotes` action
- Verify no user simulation bots are running by only starting `reference:snapshot-watch` and `liquidity:runtime`; do not run `poly-bot` simulation scripts such as `npm run dev`, `npm run sim:all`, or `npm run bots:restart-clean`

## Polygon USDC Deposits (MVP)

Automatic deposit APIs:
- `GET /api/deposits/address`
- `GET /api/deposits`
- `GET /api/admin/deposits`
- `POST /api/admin/deposits/rescan`

Monitor / worker:
- `npm run deposits:monitor`
- `npm run deposits:monitor -- --watch --pollMs 15000`

Architecture:
1. User opens `/wallet`
2. App lazily creates one Polygon USDC deposit wallet for that user
3. User sends Polygon USDC to that unique address
4. Backend monitor watches Polygon USDC `Transfer` events
5. Deposit is recorded as `DETECTED` / `CONFIRMING`
6. After `DEPOSIT_CONFIRMATIONS`, app credits ledger and `UserBalance`

Notes:
- Only Polygon network is supported
- Only Polygon native USDC is supported
- No user tx-hash submission is required
- Private keys are encrypted at rest with `DEPOSIT_WALLET_ENCRYPTION_KEY`
- Funds remain in user deposit wallets for now; treasury sweeping is a future step
