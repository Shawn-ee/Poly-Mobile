# Live Market Beta Blockers

Date: 2026-06-24

## Blocking Public Beta

- public trading is not approved
- public funding is not approved
- anonymous trading/funding must remain blocked
- no regulatory/legal readiness claim exists
- no approved live sports provider integration exists
- no settlement-grade official result source exists
- live bots are not approved

## Blocking Full Internal Live Market Beta

- no deployed event -> order -> portfolio -> final settlement drill evidence
- final settlement mutates ledger, balances, positions, orders, and market status
- no sports void/push/refund settlement path exists
- no operator evidence for repeated settlement operations
- no provider outage/stale-data drill exists

## Blocking Automated Live Sports Operation

- no approved provider credentials or contract evidence
- no provider dry-run sync implementation
- no admin-reviewed provider import flow
- no automatic suspension policy from provider data
- no result correction/stat-adjustment workflow

## Blocking Routine Settlement Use

- settlement preview is available, but final settlement remains high risk
- no void/push/refund handling for common sports outcomes
- no deployed reconciliation evidence after final settlement
- no rollback procedure can safely delete ledger/order/position data

## Required Before Upgrading Classification

To move beyond limited internal drills:

1. Run and document the controlled internal live market drill.
2. Decide whether supported sports markets exclude void/push outcomes or implement refunds.
3. Add provider-approved data sync or commit to manual-only operation.
4. Add final settlement operator runbook evidence.
5. Re-run final readiness review.
