# Deposit Wallet Security Evidence

Date: 2026-06-19

Branch: `agent/beta-deposit-wallet-security-evidence`

## Executive Summary

Phase 3B adds test and documentation evidence that controlled internal funding beta deposit wallet surfaces do not expose private wallet material and remain blocked by authentication, funding allowlist, kill switch, and deposit configuration checks.

This PR is tests/docs only. It does not change runtime funding behavior, private-key encryption, wallet generation, ledger math, deposit auto-credit, withdrawal behavior, schema, migrations, package files, workflows, deployment, or secrets.

## Evidence Added

`src/__tests__/funding-beta.routes.test.ts` now verifies:

- anonymous deposit-address access is blocked before wallet generation.
- non-allowlisted deposit-address access is blocked before wallet generation.
- deposit-address responses omit `privateKey`.
- deposit-address responses omit `encryptedPrivateKey`.
- deposit-address responses omit `seed`.
- deposit-address responses omit `mnemonic`.
- deposit-address responses do not include the synthetic test secret marker.
- funding kill switch blocks deposit-address wallet generation.
- unsafe deposit configuration blocks deposit-address wallet generation.
- production unsafe-config responses do not echo env variable names or secret markers.

Existing Phase 3 tests in `src/__tests__/funding-beta.deposit-wallet-generation.test.ts` verify:

- existing active deposit wallet reuse avoids new key generation.
- a new mocked wallet is encrypted before the Prisma create payload is built.
- raw private key is not persisted in the Prisma create payload.
- unsafe encryption config blocks wallet generation before key creation.

## Security Review

SecurityAgent result: Pass for tests/docs scope.

Private key material:

- No real private key is generated by these tests.
- Synthetic secret-like strings are used only in mocked unit tests to verify response and persistence boundaries.
- No test prints private keys, encrypted private keys, mnemonics, seed phrases, or env values.

API boundary:

- Deposit address route returns only public deposit address, network, token, minimum deposit, confirmations, and warnings.
- Private wallet material remains server-side.
- Production error responses stay generic and do not echo env names or values.

## Funding Gate Review

Funding allowlist:

- Anonymous users are blocked.
- Non-allowlisted users are blocked.
- Allowlisted access remains controlled by `INTERNAL_FUNDING_BETA_ENABLED`.

Kill switch:

- `FUNDING_KILL_SWITCH` blocks deposit-address wallet generation.

Auto-credit:

- Not changed by this PR.
- `ALLOW_AUTO_DEPOSIT_CREDIT` remains a separate explicit gate.
