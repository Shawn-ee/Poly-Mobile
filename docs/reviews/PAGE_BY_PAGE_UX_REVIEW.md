# Page By Page UX Review

## Summary

The app has the right building blocks for a sports-first prediction market, but it exposes too many parallel concepts. User-facing pages should prioritize discovery, event detail, trade review, portfolio, and account. Admin and operational pages should remain internal.

## Cross-Page Issues

- The app alternates between `PageContainer` styling and raw `main` layouts.
- Button/chip density is high for mobile.
- Beta copy is inconsistent across wallet, trade ticket, and homepage.
- Several user-facing strings include encoding artifacts.
- Empty/loading/error states should use shared components everywhere.
- Admin pages should be visibly internal and high-risk operations should be visually separated.
