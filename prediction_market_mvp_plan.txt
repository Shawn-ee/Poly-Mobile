PREDICTION MARKET MVP – FULL EXECUTION PLAN (GAME TOKEN FIRST)
PROJECT OVERVIEW

Build a Polymarket-style prediction market using GAME TOKENS (fake money).
The system must support continuous trading, early cash-out, and admin-based
market resolution. Architecture must be upgradeable to real-money trading later
without rewriting core logic.

Target:

Launch from the US

Users primarily Chinese-speaking

Game-token MVP only (no real money)

============================================================
PRODUCT GOALS

Primary Goals

Users can trade prediction markets using GAME_TOKEN

Users can cash out before market resolution

Prices update continuously via AMM

Wallet balances are derived from an immutable ledger

Admin can create, pause, and resolve markets

Leaderboard shows user performance

Non-Goals (Phase 2 only)

Real money deposits

Withdrawals

KYC / AML

Blockchain settlement

Order-book trading

LEDGER RULES (CRITICAL)

Ledger is append-only. Never update or delete entries.

Wallet balance is SUM(ledger_entries.amount_delta).

No trade may create a negative balance.

Every trade produces ledger entries.

Fees are separate ledger entries.

Settlement is deterministic and idempotent.

============================================================
TRADING MODEL (AMM)

Pricing model

LMSR-style AMM

Binary outcomes only

Price formulas
p_yes = exp(q_yes / b) / (exp(q_yes / b) + exp(q_no / b))
p_no = 1 - p_yes

BUY FLOW (ATOMIC TRANSACTION)

Confirm market is ACTIVE

Compute share cost via AMM

Check wallet balance

Insert trade record

Ledger entry: -cost (BUY)

Update AMM state

Update user position

Commit transaction

SELL FLOW (EARLY CASH-OUT)

Confirm user has enough shares

Compute proceeds via AMM

Insert trade record

Ledger entry: +proceeds (SELL)

Update AMM state

Reduce position shares

Commit transaction

FEES

Optional for MVP.
If enabled:

Fee deducted via separate ledger entry

Platform wallet receives fee

============================================================
MARKET RESOLUTION & SETTLEMENT

Admin resolves market:

Set market status to RESOLVED

Set winning outcome

For each winning position:
payout = shares * 1.0
ledger entry: +payout (WIN)

Losing positions receive nothing

Trading disabled permanently
